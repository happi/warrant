## Intent

The extension calls the server API for everything. It should read task data from local files since the repo is the source of truth. This makes it work without a server and eliminates the "not found in ledger" problem.

## Decision

Add a LocalTaskReader that parses .warrant/tasks/*.md frontmatter. Extension uses it as the primary data source. API client is optional, used only when configured (for CAS, leases). Trace comes from git log on the task file.
