## Intent

warrant task create silently fell back to local max+1 when no server/token was reachable (e.g. fresh git worktrees lack the gitignored .warrant/.env), so concurrent clients allocated colliding IDs and the server counter drifted behind git (ABS 1308 vs 1418). Make the server the single source of truth: pull IDs from the server, self-heal+retry on residual drift, and FAIL LOUDLY when a server is configured but unreachable/token-less instead of allocating a non-authoritative local ID.
